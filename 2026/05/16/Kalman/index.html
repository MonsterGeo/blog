卡尔曼滤波器

卡尔曼滤波是一种最优递归数字处理算法。为什么是最优，等一下会讲到，首先我们讲基础知识

估计、准确和精密度

一架飞机的真实位置对观察者来说是隐藏的，但我们可以使用诸如雷达一样的传感器来估计飞机的位置。每一个测量得出的参数都是一个估计。而准确度表示测量值和真实值的接近长度。

而精密度表示在同一参数下一系列测量结果的变化，准确和精密度构成了估计的基础。

一般来说，高精度系统的测量具有低方差，而低精度的测量具有高方差。随机测量误差产生了这种差异。其中，低精度系统成为有偏系统，因为它的系统内有误差。一种减小方差的方法如下：对测量值进行平均或平滑，例如我们使用温度计测量温度，由于误差随机，一些测量值将高于真实值，一些则低于真实值，那么只需要多次测量并平均，则估计值将接近真实值。

为了方便，接下来的系统我们都采用无偏的系统。

α − β − γ 滤波器

我们简单的了解一下α − β − γ滤波器，它的原理和卡尔曼滤波密切相关。

首先，我们讲一下静态系统，静态系统是指在一定时间内其状态不会发生变化的系统，比如静态系统可以是一座塔，而状态是高度。在等一下给出的例子中，我们估计金条的重量。并使用一个不会有偏差的天平。但测量包含随机噪声。

其中，系统是金条，而系统状态是金条重量。我们假定在短时间内重量不会发生变化。

在时刻n，我们记估计值x̂n, n表示先前所有测量值的平均值。那么

$$ \hat x_{n,n} = \frac{1}{n}(z_1+z_2+\cdots+z_n) = \frac{1}{n}\sum^n_{i=1}z_i $$

其中对i ∈ ℤ，每个zi表示在时刻n测出的重量。

而x̂n + 1, n表示在n时刻对n + 1时刻的估计值。在上述静态系统中，重量并不会随时间变化，因此x̂n + 1, n = x̂n, n

那么我们设计一个算法如下：

  • 根据测量值和先前的预测估计当前状态
  • 使用动态模型根据当前状态估计预测下一个状态

更改上一个式子，使得其满足我们的算法：

$$ \begin{array}{rcl} \hat x_{n,n} &=& \frac{1}{n}\sum^n_{i =1 }z_i\\ &=& \frac{1}{n}\left(\sum^{n-1}_{i=1}z_i + z_n\right)\\ &=& \frac{1}{n}\frac{n-1}{n-1}\sum^{n-1}_{i=1}z_i + \frac{z_n}{n}\\ &=& \frac{n-1}{n}\frac{1}{n-1}\sum^{n-1}_{i-1}z_i + \frac{z_n}{n}\\ &=& \frac{n-1}{n}\hat x_{n-1,n-1} + \frac{z_n}{n}\\ &=& \hat x_{n-1,n-1} - \frac{1}{n}(\hat x_{n-1,n-1}+ z_n)\\ &=& \hat x_{n-1,n-1} + \frac{1}{n}(z_n - \hat x_{n-1,n-1}) \end{array} $$

这个式子告诉我们，在时刻n − 1，对状态x的估计。这个估计的结果基于时刻n − 1的测量结果得到的。现在，我们希望得知x在n − 1时刻的时候对n的估计，即x̂n, n − 1。在本例的系统上这个状态是静态的，因此

x̂n, n − 1 = x̂n − 1, n − 1

基于上述两个关系式，我们得出状态方程

$$ \hat x_{n,n} = \hat x_{n,n-1} + \frac{1}{n}(z_n - \hat x_{n,n-1}) $$

这个方程的意思是

当前状态估计值 = 当前状态的预测值+系数乘以(测量值-当前状态的预测值)

不过注意一点，系数1/n是当前例子特有的，我们将讨论这个系数的重要性，不过在卡尔曼滤波中，这个系数的位置上的因子叫卡尔曼增益。

我们一般使用希腊字母αn 替代Kn。因此，一条更一般的状态方程如下所示：

x̂n, n = x̂n, n − 1 + αn(zn − x̂n, n − 1)

其中zn − x̂n, n − 1被称为测量误差。它是实际值和观测值之间的差。一开始这些信息还不足够告诉我们实际的状态是什么样的。所以我们使用第一次测量值$\frac{1}{n} = 1$来初始化。随着过程运行，1/n在不断减少，最终在某个测量值，误差将会几乎可以忽略不计。

在第一次测量之前，我们可以简单的阅读金条上面的标记得知大概重量。这个叫初始猜测。并且他也是我们的第一次估计。

估值算法

下述过程就是我们将会使用的估值算法

  • 步骤0 初始化：首先给出系统的状态初始值
  • 步骤1 测量： 输入测量值z0
  • 步骤2 更新： 计算卡尔曼增益α， 并使用状态更新方程估计当前状态
  • 步骤3 ： 使用模型计算下一次迭代的预测状态。并将其作为步骤2中的先验估计。

算例

第0次迭代：首先我们给出金条重量为1000g，这个值只用于初始化，在后续的迭代中就不需要用到了。此时

x̂0, 0 = 1000g

接着是第一次预测，金条的重量是不会改变的，因此动力系统是静态的，那么下一次的估计值就相当于是初始值：

x̂1, 0 = x̂0, 0 = 1000g

接着是第一次迭代，第一步测量结果如下：z1 = 996g。第二步我们计算增益，利用αn = 1/n，那么α1 = 1得到第一个观测值

x̂1, 1 = x̂1, 0 + α(z1 − x̂1, 0) = 1000 + (996 − 1000) = 996g

第三步，动力系统是静态的，因此金条重量不会变化，那么下一时刻的估计值 x̂2, 1 = x̂1, 1 = 996g

接着是第二次迭代

在一定时间后，上面计算的值称为当前迭代的先验估计值

x̂2, 1 = 996g

现在，测量值为z2 = 994g，直接第二步，计算增益 $$ \alpha_2 = \frac{1}{2} $$ 就有 x̂2, 2 = x̂2, 1 + α(z2 − x̂2, 1) = 995g

第三步，x̂3, 2 = x̂2, 2 = 995g

接下来我们给出迭代表格省略计算：

x̂n + 1, n 996 995 1003.67 1002.75 1002.6 1003.83 1000.86 997.125 996.67 999.3

α − β滤波器

现在我们考虑另一个例子，假设现在有一个雷达以每5秒为一个周期(Δt)跟踪一架飞机，在n − 1时刻，无人机的估计距离为30, 000米，且初步估计飞行速度为每秒40米。

那么利用状态更新方程，我们可以预测目标在n时刻的状态

x̂n, n − 1 = x̂n − 1, n − 1 + Δŷx − 1, n − 1 = 30000 + 5 × 40 = 30200m

其中x是距离，而y是每秒的飞行速度。

那么针对目标n时刻的速度预测为：

yn, n − 1 = yn − 1, n − 1 = 40m/s

现在，雷达的测试范围zn是30110m而不是30200m、那么预期距离和测量距离之间存在90m的误差，而造成这个误差的可能结果如下：

  • 雷达本身的测量是不准的
  • 飞机的速度发生了变化，那么新的速度值为$\frac{30110 - 30000}{5} = 22$m/s

那我们重新速度更新方程为：

$$ \hat y_{n,n} = \hat y_{n,n-1}+\beta\left(\frac{z_n - \hat y_{n,n-1}}{\Delta t}\right) $$

增益β的值依赖于雷达的精确程度。假设雷达1σ1精度为20米，那么预测距离和测量之间的误差是90米，极有可能是飞机的速度变化导致的，因此我们给增益一个较大值。若设置β = 0.9，那么 $$ \hat y_{n,n} = \hat y_{n,n-1} + \beta\left(\frac{z_n - \hat y_{n,n-1}}{\Delta t}\right) = 40 + 0.9* \frac{30110=30200}{5} = 23.8m/s $$

另一方面，若传感器一个方差内的精度为150米，那么90米的差值很可能是传感器的误差造成的，为此，β系数应该设为比较小的值，我们设β = 0.1，那么就有

$$ \hat y_{n,n} = \hat y_{n,n-1} + \beta\left(\frac{z_n - \hat y_{n,n-1}}{\Delta t}\right) = 40 + 0.1* \frac{30110=30200}{5} = 38.2m/s $$

注意的是，若飞机的速度从40m/s迅速变为22m/s，那我们在10个跟踪周期后才能看到这一变化，即上述方程迭代10次差不多。

飞机位置的状态方程和金条问题中推出的差不多，都是 x̂n, n = x̂n, n − 1 + α(zn − x̂n, n − 1)

像前一个例子（α每次迭代重新计算）不同，这里的α是一个常数。

α的大小取决于雷达的精度，对于高精度的雷达，我们选择较大的α，这里α = 1表示估计距离等于实际距离

若α = 0，则更新方程给出测量值没任何意义。

所以，我们推导出构成雷达跟踪器状态更新方程的非操作，他们也叫α − β轨迹滤波方程。

即：

$$ \begin{array}{rcl} \hat x_{n,n} &=& \hat x_{n,n-1}+\alpha(z_n - \hat x_{n,n-1})\\ \hat y_{n,n} &=& \hat y_{n,n-1}+\beta\left(\frac{z_n - \hat x_{n,n-1}}{\Delta t}\right) \end{array} $$

α − β估值算法

依旧5个模块互相作用，首先是初始化模块，第1步，我们给系统一个猜测的初始值。接着，我们再获取测量值。测量值送入更新器，更新器接收α, β参数更新状态方程。然后输出到预测模块。预测模块输出对下一时刻的预测值作为下一个时刻的先验估计。

现在，我们给两个α − β的参数为 $$ \begin{aligned} \alpha = 0.2\\ \beta = 0.1 \end{aligned} $$

追踪周期为5秒一次。

给定0时刻的初始值为x̂0, 0 = 30000m和ŷ0, 0 = 40m/s
